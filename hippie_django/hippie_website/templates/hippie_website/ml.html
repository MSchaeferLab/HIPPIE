A PPI-Prediction Workflow

A typical end-to-end example: choose data, pull it out of HIPPIE, train two contrasting models (a classic feature-based baseline and a sequence-based model), and evaluate honestly. The running theme — taken from the papers above — is that how you split your data matters more than which model you pick.

{# ── Step 1 ── #}
1

How do I choose my data?

A PPI classifier needs positive examples (interacting pairs) and negative examples (non-interacting pairs). HIPPIE gives you scored positives directly; negatives are the hard part.

  • Positives — interactions above a confidence threshold. Higher thresholds (≥ {{ current_release.int_q3|default:0.00|floatformat:2 }}, high confidence) trade quantity for reliability.
  • Negatives — there is no gold-standard "these never interact" set. Common choices: random non-edges, or curated non-interactions. Random negatives are biased by node degree, which a model can exploit instead of learning biology.
  • Scope — restrict to a tissue or a confidence band so the question is well-defined (e.g. "high-confidence PPIs among colon-expressed proteins").
Leakage trap (paper 1). If the same protein appears in both train and test — or two near-identical sequences are split across train/test — the model can memorise per-protein or per-family signal and report inflated accuracy. Decide your split strategy before sampling negatives.
{# ── Step 2 ── #}
2

How do I pick that in HIPPIE?

Use the ML Splits tool. It applies the same filters as Browse (confidence score, source, experiment type, interaction type, tissue / RPKM, node degree) and then builds a labelled train/test dataset with negative sampling for you:

  • Set min_score (e.g. {{ current_release.int_q3|default:0.00|floatformat:2 }}) to keep high-confidence positives.
  • Add a tissue filter and min_rpkm to scope the proteome.
  • Choose a neg_ratio (negatives per positive) and a fixed seed so the export is reproducible.
  • Decide whether to include_isoforms.

The tool returns a downloadable TSV with labelled pairs. You can also drive it from the JSON API directly: 

# Create a split job, then poll for the result, via the HIPPIE API
import requests, time

BASE = "https://your-hippie-host"          # this server
params = {
    "min_score": {{ current_release.int_q3|default:0.00|floatformat:2 }},                       # high-confidence positives only
    "tissue_ids": [5],                       # e.g. colon
    "min_rpkm": 1.0,
    "neg_ratio": 1.0,                        # balanced negatives
    "include_isoforms": False,
    "seed": 78539105873,                     # reproducibility
}
job = requests.post(f"{BASE}/api/browse/splits/", json=params).json()
job_id = job["job_id"]

# poll until the background job finishes
while True:
    status = requests.get(f"{BASE}/api/browse/splits/{job_id}/").json()
    if status["state"] in ("done", "failed"):
        break
    time.sleep(2)

# download the labelled train/test TSV
tsv = requests.get(f"{BASE}/api/browse/splits/{job_id}/download/").content
open("hippie_ppi.tsv", "wb").write(tsv)

How do I create a model with the data?

We build two models side by side: a classic feature-based baseline and a sequence-based model. Both share the same split so the comparison is fair.

Load the data 
import pandas as pd

df = pd.read_csv("hippie_ppi.tsv", sep="\t")
# columns: protein_a, protein_b, label (1 = interacts, 0 = negative)
print(df["label"].value_counts())
A — Classic baseline (scikit-learn)

Cheap topological features straight from the HIPPIE network. Per paper 1, this kind of baseline is surprisingly hard to beat — always report against it. 

# pip install scikit-learn networkx
import networkx as nx
from sklearn.ensemble import RandomForestClassifier

# build the graph from POSITIVE training edges only (no test leakage!)
train = df[df.split == "train"]
G = nx.Graph()
G.add_edges_from(train[train.label == 1][["protein_a", "protein_b"]].itertuples(index=False))

def features(a, b):
    deg_a = G.degree(a) if a in G else 0
    deg_b = G.degree(b) if b in G else 0
    common = len(list(nx.common_neighbors(G, a, b))) if a in G and b in G else 0
    return [deg_a, deg_b, deg_a * deg_b, common]

X_train = [features(a, b) for a, b in train[["protein_a", "protein_b"]].itertuples(index=False)]
clf = RandomForestClassifier(n_estimators=300, n_jobs=-1, random_state=0)
clf.fit(X_train, train["label"])
B — Sequence-based model

Embed each protein sequence with a protein language model (e.g. ESM-2), concatenate the pair, and train a small classifier. More expressive — and far more prone to leakage. 

# pip install torch fair-esm scikit-learn
import torch, esm
from sklearn.linear_model import LogisticRegression

model, alphabet = esm.pretrained.esm2_t12_35M_UR50D()
batch_converter = alphabet.get_batch_converter()
model.eval()

def embed(seq):
    _, _, toks = batch_converter([("p", seq)])
    with torch.no_grad():
        out = model(toks, repr_layers=[12])["representations"][12]
    return out[0, 1:len(seq) + 1].mean(0).numpy()   # mean-pooled residue embeddings

# seqs: dict protein_id -> amino-acid sequence (fetch from UniProt)
emb = {p: embed(seqs[p]) for p in set(df.protein_a) | set(df.protein_b)}

import numpy as np
def pair_vec(a, b):
    return np.concatenate([emb[a], emb[b]])

Xtr = np.array([pair_vec(a, b) for a, b in train[["protein_a", "protein_b"]].itertuples(index=False)])
seq_clf = LogisticRegression(max_iter=1000).fit(Xtr, train["label"])
Where leakage creeps in (paper 2, Q on data splitting). A random pair split lets the same protein sit in both train and test. The sequence model then just recognises proteins it has already seen — not interactions. A naive random split can push AUROC from ~0.6 to >0.9 of pure illusion.
Fix. Split on proteins, not pairs: assign each protein to train or test, and cluster by sequence similarity (e.g. MMseqs2 / DataSAIL) so near-identical proteins never straddle the split. The ML Splits tool's seed keeps this reproducible; re-cluster if you change the protein set.

4

How do I evaluate this?

Score both models on the held-out test split and always compare against the baseline. Because negatives usually outnumber positives, report AUPRC alongside AUROC. 

from sklearn.metrics import roc_auc_score, average_precision_score

test = df[df.split == "test"]
y = test["label"].to_numpy()

Xte_base = [features(a, b) for a, b in test[["protein_a", "protein_b"]].itertuples(index=False)]
Xte_seq  = np.array([pair_vec(a, b) for a, b in test[["protein_a", "protein_b"]].itertuples(index=False)])

for name, proba in [
    ("baseline", clf.predict_proba(Xte_base)[:, 1]),
    ("sequence", seq_clf.predict_proba(Xte_seq)[:, 1]),
]:
    print(f"{name:9s}  AUROC={roc_auc_score(y, proba):.3f}  "
          f"AUPRC={average_precision_score(y, proba):.3f}")
  • Beat the baseline, or report why not. If the sequence model only wins under a random split but ties the baseline under a similarity-aware split, the "win" was leakage.
  • Stratify by novelty. Report metrics separately for test proteins that are dissimilar to anything in training — that is the realistic use case.
  • Fix the seed and publish the split. Reproducibility is part of the result.

For the full reasoning behind each of these checks, see the two papers under Key Reading above.
