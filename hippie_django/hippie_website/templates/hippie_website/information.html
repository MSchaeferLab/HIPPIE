{# ── Welcome ── #}

Welcome to HIPPIE

Welcome to HIPPIE (the Human Integrated Protein Protein Interaction rEference)! We provide confidence scored and functionally annotated human protein-protein interactions (PPIs). This page contains details on the information stored in HIPPIE and the multiple ways to access it. First time users might want to start with the walk-through example to get an idea what types of analyses can be performed with HIPPIE.

{# ── Confidence scoring ── #}

Confidence scoring of experimentally measured interactions

A core component of HIPPIE is the confidence scoring of interactions based on the amount and reliability of evidence supporting them.

FORMULA

This score is calculated as a weighted sum of the number of studies in which an interaction was detected, the number and quality of experimental techniques used to measure an interaction and the number of non-human organisms in which an interaction was reproduced.

{# ── Query interactions ── #}

Query interactions

The query mode allows for querying HIPPIE with one or more proteins (up to 50, separated by comma, space or tab), each given as a UniProt identifier (id or accession), gene symbol, Entrez gene id or Ensembl id (ENSG/ENST). All interactions in HIPPIE that match any of the query proteins are displayed together.

If you want to query specific interactions directly, you may do so in the interaction query.

On the result page, UniProt accession, Entrez gene id and gene symbol of each interactor of the query are indicated. The confidence score of the interaction is given in the last column. Clicking on the arrow button or the row opens an evidence page which lists the source databases, the publications which mention the interaction, the experiment type used to detect it, and the species in which it is conserved.

{# ── Network construction ── #}

Network construction

In the network construction mode, several proteins can be used as seed proteins to construct a first-order network. The resulting network can be visualized graphically, listed in a tabular format in the browser or downloaded as a tsv file.

Proteins and interactions submitted by the user are highlighted in blue in our graphical output and, in case they are not in HIPPIE, they are highlighted in red. Several filter criteria allow to modify the resulting network (see Filters section). Note, however, that these filters are only applied to interactions added from the HIPPIE repository and do not affect the input proteins or interactions, which will be displayed regardless of whether they pass the filter criteria or not.

HIPPIE offers to display any constructed subnetwork with the light-weight network visualization tool Cytoscape.js.

{# ── Browsing ── #}

Browsing the HIPPIE proteome.

The browsing mode in HIPPIE allows for the exploration of the entire set of proteins or interactions in the database. Note that loading the entire HIPPIE proteome may take a few seconds, depending on the type of Internet connection being used.

Download the repository

The HIPPIE dataset is provided in two different download formats: PSI-MITAB 2.5 and our own tab-separated flat file format. Besides offering the data in compliance with the specification of the Proteomics Standards Initiative (PSI), we decided to additionally offer the data in a format described on the download page.

{# ── HIPPIE API ── #}

HIPPIE REST API

HIPPIE can be queried programmatically through a small set of JSON endpoints, so it can be integrated into any bioinformatics pipeline. All paths below are relative to the HIPPIE base URL and return application/json unless noted otherwise. Parameters marked (repeatable) may be supplied more than once (e.g. &source=1&source=2).

Shared filter parameters (accepted by most endpoints):

  • show = interactions, noninteractions or both.
  • min_score / max_score = confidence-score bounds (0–1).
  • source, experiment, interaction_type, tissue = numeric ids (repeatable).
  • min_rpkm = minimum tissue expression (used together with tissue).
  • min_degree, min_avg_score = protein-level thresholds.
  • reviewed = reviewed, unreviewed, or both.
  • isoform_mode = general, isoforms, or both.

Protein query

GET /api/query/?q=TP53,MDM2,BRCA1&show=both&min_score=0.63

Interactions of one or more proteins. q accepts up to 50 identifiers (separated by comma, space or tab) — each a gene symbol, UniProt accession or entry name, Entrez id, isoform accession, or Ensembl id (ENSG/ENST). Each is resolved independently and the interactions are returned as one combined list; the shared filters apply. Returns { query_proteins, isoforms_included, expanded_proteins, unresolved, interactions[], error }, where unresolved lists any inputs that matched nothing (the request still succeeds for the rest).

Batch pairwise lookup

POST /api/interaction/   (JSON body)

Looks up the recorded interaction for each supplied pair. Body: { "pairs": [ { "a": "TP53", "b": "MDM2", "input_order": 0 } ], … filters } (up to 200 pairs per request). Returns { results: [ … ] }; a row with score = -1.0 means that pair was not found.

Network expansion

POST /api/network/   (JSON body)

Builds the first-shell sub-network around a set of seed proteins (every edge touching a seed). Body: { "proteins": "TP53 BRCA1 ATM", … filters } (identifiers separated by whitespace, commas or semicolons). Returns { node_count, edge_count, interactions[], seed_ids, unresolved, truncated, total_edges, error }.

Browse (paginated)

GET /api/browse/proteins/?offset=0&limit=50&sort=degree&dir=desc
GET /api/browse/interactions/?offset=0&limit=50&show=both

/api/browse/proteins/ returns a page of proteins { total, proteins[] } (each protein carries degree, avg_score and is_reviewed); /api/browse/interactions/ returns a page of interactions { total, interactions[] }. Both accept offset, limit (≤ 200), a free-text search field q (up to 50 identifiers separated by comma, space or tab — matches are unioned), sort/dir, and the shared filters.

Bulk export (TSV)

GET /api/browse/export/?mode=interactions&show=both

Streams the filtered result set as a tab-separated file (mode=proteins or mode=interactions, using the same filters as the matching browse endpoint). Capped at 50 000 rows; the response sets the header X-Export-Truncated: 1 when the cap is reached.

{# ── Filters ── #}

Filters

The query, interaction, network and browse pages share one filter panel (each page shows the subset of controls relevant to it). Filters take effect only when you run or re-apply the search, and they combine with logical AND — a result must pass every active filter to be shown. On the network construction page they restrict only the edges pulled from HIPPIE, never your own input proteins or interactions.

  • Result type — show interactions, non-interactions (protein pairs experimentally reported not to interact, drawn on a grey background), or both.
  • Confidence score — keep interactions whose score lies between a minimum and maximum bound (0–1). The medium and high presets raise the minimum to the current release's median (Q2) and upper-quartile (Q3) score for the selected result type.
  • Source database — keep interactions reported by any of the selected source databases (see the Sources section).
  • Experiment type — keep interactions detected by any of the selected experimental methods.
  • Interaction type — keep interactions classified as any of the selected interaction types.
  • Tissue expression — keep interactions whose proteins are expressed in any of the selected tissues; an optional minimum median RPKM raises the required expression level.
  • Protein filters — restrict to proteins by degree (the number of interactions touching a protein) and by average score (the mean confidence across those interactions).
  • Review status — reviewed (manually curated UniProtKB/Swiss-Prot entries), unreviewed (automatically annotated UniProtKB/TrEMBL entries), or both.
  • Isoforms — general (canonical proteins; an isoform appears only when you query it explicitly), isoforms (interactions with at least one non-canonical isoform partner), or both. See the Isoforms section below.

{# ── Isoforms ── #}

Isoforms

A single gene can give rise to several protein isoforms through alternative splicing. HIPPIE identifies an isoform by its UniProt isoform accession (for example P37163-2), in comparison to the plain accession (P37163). On an interaction's detail page the canonical isoform of a gene is marked with a canonical chip; canonical status is determined by which isoform is referencec by UniProt to have the same UniParc sequence as the general protein.

The three-way Isoforms filter controls which of these are shown: general keeps only proteins, isoforms keeps only interactions in which at least one partner is an isoform, and both filters both to connected proteins and isoforms.

To provide isoform- and domain-resolved interaction information, each interaction's detail page cross-links to DIGGER, which maps protein-protein interactions to the specific domains and isoforms that mediate them.
