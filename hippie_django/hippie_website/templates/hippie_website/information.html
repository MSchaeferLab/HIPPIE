{# ── About ── #}
About HIPPIE

HIPPIE (Human Integrated Protein–Protein Interaction rEference) is a scored, integrated resource of human protein–protein interactions (PPIs). It aggregates experimental interaction data from multiple public databases and assigns a unified confidence score to each interaction based on the quality and quantity of supporting evidence.

HIPPIE is maintained at ExB / TU München. It is freely available for academic use.

{# ── Confidence scoring ── #}
Confidence Scoring

Each interaction is assigned a score between 0 and 1 based on a weighted sum of experimental evidence. The formula accounts for the quality of the detection method, the number of independent studies, and cross-species conservation.

score(A, B) = norm( Σe w(e) × n(e) + conservation_bonus )
where:
  w(e) = quality weight of experiment type e (see table below)
  n(e) = number of independent studies using method e
  conservation_bonus = 0.1 × (number of orthologous species)
Score thresholds
≥ 0.72
High confidence
Supported by multiple independent studies using high-quality methods (e.g. X-ray crystallography, SPR).
≥ 0.63
Medium confidence
Supported by at least one well-validated method or multiple lower-quality studies.
< 0.63
Low confidence
Single study or low-quality detection method. Use with caution.
{# ── Data sources ── #}
Data Sources

HIPPIE integrates interaction evidence from the following databases:

BioGRID General repository thebiogrid.org
IntAct EBI curated database ebi.ac.uk/intact
MINT Molecular interactions mint.bio.uniroma2.it
HPRD Human proteome reference hprd.org
DIP Database of interacting proteins dip.doe-mbi.ucla.edu
HomoMINT Predicted human interactions mint.bio.uniroma2.it
{# ── Querying ── #}
Querying HIPPIE
Accepted identifier formats
  • Gene symbol — e.g. BRCA1, TP53
  • UniProt entry ID — e.g. BRCA1_HUMAN
  • UniProt accession — e.g. P38398
  • Entrez Gene ID — e.g. 672
Query types
  • Protein Query — all interactions for a single protein
  • Interaction Query — look up specific protein pairs in bulk
  • Network Query — extract a sub-network from seed proteins
  • Browse — explore all proteins with filtering
{# ── REST API ── #}
REST API

All query functions are available programmatically via JSON endpoints.

GET
/api/query/?q=BRCA1
Returns all interactions for a protein. Accepts any identifier format.
POST
/api/interaction/
Look up specific protein pairs. Body: {"pairs": [{"a": "BRCA1", "b": "TP53", "input_order": 0}]}
GET
/api/browse/?offset=0&limit=500&tissue=5&min_score=0.63
Paginated protein list with server-side filtering by tissue, source, degree, and score.
GET
/api/browse/filters/
Returns available tissue and source database lists for filter dropdowns.
